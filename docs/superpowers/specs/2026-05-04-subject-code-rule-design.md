# 科目编码规则与树形科目设计

## 一、财务专业业务规则

科目必须遵循账套统一编码规则，默认规则为 `4-2-2-2`。一级科目编码长度为 4 位，二级在一级基础上追加 2 位，三级继续追加 2 位，四级继续追加 2 位。科目编码只能使用数字。新增下级科目时，编码必须以上级科目编码作为前缀，并且长度必须等于上级级次的下一档长度。非末级科目不能直接录入凭证；已有期初余额或凭证的科目不能随意修改科目类别和层级。

## 二、数据库表结构

在 `fin_account_set` 增加 `subject_code_rule varchar(50) not null default '4-2-2-2'`，作为账套级科目编码规则。该字段随账套参数保存，所有科目新增与编辑校验均读取当前账套规则。`fin_subject` 继续保存 `subject_code`、`parent_code`、`level_no`、`leaf_flag`、`voucher_entry_flag` 等字段，不新增冗余层级表。

## 三、业务流转流程

用户进入“基础资料-科目”后，前端读取科目列表和编码规则。科目按六大类页签展示，当前类别内按编码树形展开。用户点击“编码设置”后修改追加级长，保存到当前账套。用户新增科目时，前端根据所选父级提示应录入的编码长度和前缀；提交后后端再次校验，校验通过才写入科目。

## 四、后端接口清单

`/subject/list` 返回当前账套科目列表，按 `subject_code` 排序。`/subject/codeRule` 返回当前账套编码规则。`/subject/codeRuleSave` 保存当前账套编码规则。`/subject/add` 和 `/subject/save` 复用统一编码校验。

## 五、枚举/字典/基础档案设计

科目类别沿用六大类：资产 `ASSET`、负债 `LIABILITY`、共同 `COMMON`、所有者权益 `EQUITY`、成本 `COST`、损益 `PROFIT_LOSS`。编码规则由整数段组成，首段固定显示为一级科目位数，后续段为每级追加位数，默认 `4-2-2-2`。

## 六、实现方案

前端科目页改为顶部六大类页签，表格使用 Element Plus 树表格，编码列按层级缩进并显示展开状态。工具条增加“编码设置”按钮，弹窗使用 `4 - 2 - 2 - 2` 形式编辑规则。后端在 `Subject` 模型中增加规则解析、级次推导、父级合法性校验和账套规则读写。数据库 DDL、预置数据和升级补丁同步增加 `subject_code_rule`。
